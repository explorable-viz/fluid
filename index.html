fluid:
functional language for understanding and interacting with dependencies

Fluid utilises program analysis/data provenance techniques for connecting visualisations to data sources in a fine-grained way. For example, our proof-of-concept implementation can turn a visual selection in a bar chart into a "data selection" (a restricted view of a dataset where only relevant rows are shown, and relevant cells within that are highlighted):

This data linking feature can also be used to link visualisations to other visualisations, something possible with some effort and/or restrictions with existing tools, but which our approach provides automatically:

As well as offering the first fully automated (and thus potentially pervasive) solution to this well-recognised problem in data visualisation, our approach to data linking is "transparent", in that a view of the specific data elements which "explain" why the two visual elements are linked can always be made available. Over the next year we plan to extend this into a mature toolkit for authoring data-linked visualisations for use in data journalism and science publishing, enabling online articles with visual components which are connected automatically to the data sources being visualised.
